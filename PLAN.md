# 智慧学生管理系统 — 统一开发计划

> **核心原则：模块优先，角色次之**
> 每个功能模块包含学生端/教师端/管理端三端视图，共享同一数据层。
> 教师端细分为：**班主任**（看所有科目）vs **科任老师**（只看自己教的科目）。
> 管理端只看全局概览，不做深度分析（深度分析留给 AI）。

## 📌 已完成模块

### ✅ 成绩模块
* **数据层**：`exams`（考试）+ `scores`（成绩），按班级+科目+学生关联，排名（DENSE_RANK）在保存时自动重算。
* **学生端**：成绩趋势图（Chart.js 柱状图+折线图，支持切换科目）、雷达图（对比本次/上次能力）、班级排名（标示自己位置及排名变动）以及详细成绩表。
* **教师端（科任老师）**：选班-选考试-选科目三级筛选，统计卡片，可编辑行内成绩表，成绩自动重算排名，新建考试，本科目历次成绩均分对比。
* **教师端（班主任）**：切换查看本班所有科目成绩，跨科目平均分趋势对比。
* **管理端**：全校成绩均分及及格率对比，一键导出为 Excel 兼容的 UTF-8 BOM CSV 报表。

### ✅ 作业模块 (4.2.1)
* **学生端**：作业列表分组展示（未提交/已提交/已批改），在线提交作业内容，查看评分与评语。
* **教师端**：选班级发布作业，实时监控班级提交进度，打分与写评语。
* **管理端**：全校作业监控大屏，高亮标红提交率低于 70% 的异常班级。

### ✅ 考试安排模块 (4.2.2)
* **学生端**：排考日程时间轴展示，已结束考试一键直达成绩页。
* **教师端**：所管辖班级考试日程的增删改查管理，快捷设置时间地点。
* **管理端**：全校多班级排考批量发布，排考冲突检测预警面板。

### ✅ 错题本模块 (4.2.3)
* **学生端**：录入错题（题干、我的答案、解析等），错题待复习/已掌握分组管理，错题总量及掌握率分析。
* **教师端**：班级错题分析（班级总错题数、高频错题 Top 5 及错题学生分布排行）。

### ✅ 课表模块 (4.2.4)
* **学生端**：5×8 节彩色周课表网格，今日课程高亮展示，支持点击显示教师姓名。
* **教师端**：我的课表（跨班授课展示），本班完整课表切换查看。
* **管理端**：全校排课管理工具，批量排课与保存。

### ✅ 考勤与请假模块 (4.4)
* **数据层**：`attendance`（考勤明细）+ `leave_requests`（请假单），请假审批同意后自动同步考勤。

* **教师端**：选班级和课节一键点名录入；请假审批流转。
* **管理端**：全校考勤总览统计，严重缺勤风险预警。

### ✅ 教师端教学增强 (4.3)

### ✅ 验证码模块 (自定义)

## 🏗️ 技术架构要点

### 数据层共享原则
```
同一张表 → 同一套 API → 不同角色通过权限看到不同数据

例：scores 表
  - 学生调用 GET /scores/my    → 只看到自己
  - 科任老师调用 GET /scores/class-scores?subject=math → 只看到 math
  - 班主任调用 GET /scores/class-scores → 看到本班所有科目
  - 管理员调用 GET /scores/export → 全校成绩导出
```

### 权限模型
```
学生 (user_type=student)
  └── 只能查看自己的数据

科任老师 (user_type=staff, subject=具体科目)
  └── 可查看/编辑所教科目的所有班级数据

班主任 (user_type=staff, homeroom_teacher_id 匹配)
  └── 可查看/编辑本班所有科目的数据

管理员 (user_type=staff, role=admin)
  └── 所有数据的全局视图
```
