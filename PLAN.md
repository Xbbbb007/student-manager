# 智慧学生管理系统 — 统一开发计划

> **核心原则：模块优先，角色次之**
> 每个功能模块包含学生端/教师端/管理端三端视图，共享同一数据层。
> 教师端细分为：**班主任**（看所有科目）vs **科任老师**（只看自己教的科目）。
> 管理端只看全局概览，不做深度分析（深度分析留给 AI）。

---

## 📌 已完成模块

### ✅ 成绩模块

#### 数据层
- 表：`exams`（考试）+ `scores`（成绩），按班级+科目+学生关联
- 排名：DENSE_RANK（同分并列），保存时自动重算

#### 学生端
- **成绩趋势图**：柱状图（分数）+ 折线图（班排名+校排名），支持按科目切换
- **雷达图**：五科能力雷达，可切换「本次/上次」对比
- **班级排名**：显示自己前后各3名，标注自己位置，展示排名变化和分数变化
- **详细成绩表**：每次考试各科成绩 + 涨跌标注

#### 教师端（科任老师）
- **选班 → 选考试 → 选科目** 三级筛选
- **统计卡片**：平均分、最高分、最低分、及格率、优秀率
- **可编辑成绩表**：行内输入分数，保存后自动重算排名
- **新建考试**：指定班级+日期
- **本科目对比**：查看自己所教科目在历次考试中的均分变化

#### 教师端（班主任）
- 以上全部，但可切换查看**所有科目**（不限于自己教的）
- 可以看到**跨科目对比**（如语文平均分 vs 数学平均分趋势）

#### 管理端
- **全校概览**：各年级/各班平均分对比、及格率排名
- （后期接入 AI 后可查看中位数、方差、预警等深度分析）


#### 教师端（班主任）
- 以上全部，且可查看**所有科目**的作业情况
- 班级维度统计：哪些学生总提交率低、哪些科目作业积压多

#### 管理端
- **全校作业概况**：各班各科作业布置量、提交率概览

## 🏗️ 技术架构要点

### 数据层共享原则
```
同一张表 → 同一套 API → 不同角色通过权限看到不同数据

例：scores 表
  - 学生调用 GET /scores/my    → 只看到自己
  - 科任老师调用 GET /scores/class-scores?subject=math → 只看到 math
  - 班主任调用 GET /scores/class-scores → 看到所有科目
  - 管理员调用 GET /scores/overview → 全校概览
```

### 权限模型
```
学生 (user_type=student)
  └── 只能查看自己的数据

科任老师 (user_type=staff, subject=具体科目)
  └── 可查看/编辑所教科目的所有班级数据

班主任 (user_type=staff, homeroom_teacher_id 匹配)
  └── 可查看/编辑本班所有科目的数据

管理员 (user_type=staff, role=admin)
  └── 所有数据的全局视图
```
